# Lab 00 Design Spec
**Date:** 2026-04-07
**Author:** Dr. Patrick Bitterman

---

## Overview

Lab 00 is an introductory lab designed to precede Lab 01. Its purpose is to get students fully set up with R, RStudio, GitHub, and GitHub Desktop, and to verify that setup by completing a series of basic R tasks. It is not a "mini lab" — it is a full introductory lab, just without spatial content.

**Audience:** Near-zero R experience, almost no GitHub experience. A small number of more experienced students may be present.

**Deliverable:** A completed `.Rmd` document, knitted to PDF, pushed to a public GitHub repository. Students submit the repo link on Canvas — same workflow as all subsequent labs.

---

## Philosophy & Format

- Follows the same structure and tone as Lab 01–06
- YAML header matches existing labs
- Instructional walkthrough first (with embedded code chunks students can run), then a formal "Your Tasks" block, then a "Questions" block
- Students are expected to insert their own chunks during the tasks section — this is explicitly practiced

## Lab Sections

### Preamble
Standard formatting/submission instructions, identical in spirit to Lab 01:
- Public GitHub repo required
- Canvas link submission before due date
- Code in `.Rmd` format (required for this lab, unlike later labs which allow `.R`)
- Relative file paths required for any data

### 1. Getting Set Up
Step-by-step instructions for:
- Installing R (CRAN)
- Installing RStudio
- Creating a GitHub account
- Installing GitHub Desktop
- Creating a new public repository on GitHub
- Cloning the repo locally via GitHub Desktop
- Opening the `.Rmd` lab file inside the cloned repo in RStudio

This section is prose/checklist — no code chunks.

### 2. Your First RMarkdown Document
Explain the three components of an `.Rmd` file:
- YAML header (metadata)
- Prose (markdown text)
- Code chunks (R code between ` ```{r} ``` ` delimiters)

Walk through knitting to PDF. Then ask students to insert a new chunk manually (using the RStudio "Insert Chunk" button or keyboard shortcut) and write a simple expression in it (`1 + 1`), then re-knit to confirm it appears in the output.

### 3. Basic R Operations
Walkthrough covering:
- Assigning variables with `<-`
- Arithmetic operators (`+`, `-`, `*`, `/`, `^`)
- Data types: `numeric`, `character`, `logical`
- Checking types with `class()` and `is.numeric()` / `is.character()`

### 4. Vectors
Walkthrough covering:
- Creating vectors with `c()`
- Arithmetic on vectors (element-wise)
- Common summary functions: `sum()`, `mean()`, `length()`, `min()`, `max()`

### 5. Packages
Walkthrough covering:
- What a package is
- `install.packages("packagename")` — done once
- `library(packagename)` — done each session
- Install and load `tidyverse` as the worked example

### 6. Reading and Summarizing Data
Walkthrough covering:
- Reading a CSV with `read_csv()` (from tidyverse)
- `head()`, `tail()`, `summary()`, `glimpse()`
- Brief discussion of what each function tells you

**Data:** A simple, non-spatial CSV already in the course repo. Suggested: a plain tabular dataset from the `/data/` directory, or a small purpose-built CSV (a few columns, ~50–100 rows) added specifically for this lab.

### 7. Basic Plotting
Walkthrough covering:
- Base R `hist()` for a single numeric variable
- Adding axis labels (`xlab`, `ylab`) and a title (`main`)

Note: `ggplot2` is intentionally deferred to Lab 01 and beyond. Base R plotting is introduced here as the simplest possible entry point.

## Tasks Block

Students complete all tasks in their `.Rmd` document. Each task requires R code AND, where indicated, explanatory comments. Tasks must be clearly labeled.

### Task 1: R as a Calculator
- 1.1 Create a variable storing a number representing your (fake/approximate) age in years. Convert it to days and print the result.
- 1.2 Create two numeric vectors of length 5. Add them together, then compute the `mean()` and `sum()` of the result.

### Task 2: Working with Data Types
- 2.1 Create a character vector of 4 U.S. state names and a logical vector indicating whether each state is in the Midwest. Print both.
- 2.2 Attempt to perform arithmetic on a character vector. Show the error output and write a comment explaining what happened and why.

### Task 3: Packages
- 3.1 Install and load `tidyverse`. Write a comment in your code explaining the difference between `install.packages()` and `library()`, and why calling `install.packages()` every time you run your script is a bad idea.
- 3.2 Run `sessionInfo()` and leave the output visible in your knitted document.

### Task 4: Reading and Summarizing Data
- 4.1 Read the provided CSV file using `read_csv()`. Run `head()`, `tail()`, and `summary()` on it. Write a comment for each describing what it tells you about the data.
- 4.2 Use `glimpse()` to inspect the data. In a comment, report how many rows and columns the dataset has.

### Task 5: RMarkdown Practice
- 5.1 Insert a new code chunk (do not copy-paste — use the RStudio insert chunk button or keyboard shortcut). In that chunk, write code that prints the message `"Lab 00 complete"`. Knit your document and confirm the output appears in the PDF.

### Task 6: Basic Plotting
- 6.1 Make a histogram of one numeric column from the CSV using `hist()`. Label your axes and give the plot a title.
- 6.2 Make a histogram of a different numeric column using `hist()`. Adjust the number of bins using the `breaks` argument.

---

## Questions Block

1. What is the difference between `install.packages()` and `library()`? Why would it be a problem to call `install.packages()` every time you run your script?
2. What does a relative file path mean, and why does this lab require them instead of absolute paths?
3. What did you find confusing or surprising about RStudio, GitHub, or R itself? What questions do you still have?

---

## Data Requirements

A simple, non-spatial CSV is needed for Tasks 4 and 6. It should have:
- At least 2–3 numeric columns suitable for histograms
- At least 1 character/categorical column
- ~50–100 rows
- No spatial attributes

Options:
- A purpose-built CSV added to `/data/lab00/` in the course repo
- An existing non-spatial CSV already in the repo (e.g., from CBW data)

The lab instructions should point to this file using a relative path consistent with the `.Rmd` file location in the repo.
